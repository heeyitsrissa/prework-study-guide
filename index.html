HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Using the head element we can use metadata tags such as the title, style, base, link, meta,and script.
  • The title element is important not only to title the webpage but is required for Search Engine Optimization.
  • The style defines the styles in CSS related to a website.
  • The base element describes the base URL for a webpage.
  • The link element joins the HTML document to an outside resource.
  • The meta element defines metadata like the viewpoint, keywords, character set, author, and description.
  • The Script element inserts client-side scripts in the HTML document.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is used to style directly in the HTML file, it is useful for quick changes but not very flexible because each inline style must be edited individually.
  • Internal style sheet contains CSS rules for webpage in the head section of the HTML file. Rules apply only for that page, you can identify elements using CSS classes and IDs to style multiple elements in the page.
  • External CSS style sheet is the most common want to style a webpage is preferred because it allows you to keep CSS rules in a seperate file making design changes easier.
  • Declarations are the CSS properties applied and the values of the properties.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications from the current working branch to the staging area which tells Git changes will be added in the next commit.
  • git commit -m: is a command written with a commit message that explains what the commit contains, it is a snapchot of the current changes.
  • git pull origin main: Checks that our local branch is in sync with the base branch in GitHub.
  • git push origin branch-name: pushes changes to the remote branch.
  • A repository or 'repo' is a folder for your project that contains all versions and filesof the project.
  • A branch is a workspace where changes can be made without affecting the live site.
  • To commit changes means whenever you add, edit, or delete a file you'remaking a commit.
  • A pull request or'PR' means you want the changes made on a branch to merge into another branch, typically a way to review and discuss your workwith someone else.
  • When a pull request is approved changes will merge into a new branch.
  • An issue is a way to report a new task you are working on and track the progress.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a single variable that is used to hold a group of related data.
  • Control flow can be interupted by using conditional statements and loops.
  • A function is a set of instructions to tell the computer how to perform task.
